# B-Equip 가먼트 프린터 커맨드라인 도구 분석

> 분석일: 2026-03-18
> 원본: BCMD_Commandline_Ver.2.6.0_E.pdf, BCMD_Commandline_Ver.2.6.0_ChangeList_E.pdf

## 1. 개요

### 1-1. 장비 정보

- **장비명**: B-Equip (B-422)
- **유형**: 가먼트(의류) 프린터
- **연동 방식**: Windows 커맨드라인 도구 (`BCMDCMD.exe` + `BCMDApi.dll`)
- **지원 OS**: Windows 7/8.1/10 (32/64bit)
- **필수 조건**: BCMD 프린터 드라이버 설치 필요

### 1-2. 커맨드라인 도구란

이미지 파일과 인쇄 설정을 지정하여 ARX4 형식의 인쇄 데이터를 생성하고, 프린터에 전송하며, 프린터 상태를 조회할 수 있는 CLI 애플리케이션이다.

### 1-3. 파일 구성

| 파일          | 설명                                   |
| ------------- | -------------------------------------- |
| `BCMDCMD.exe` | 커맨드라인 실행 파일                   |
| `BCMDApi.dll` | API 라이브러리 (동일 폴더에 배치 필요) |

---

## 2. 주요 명령어 (Commands)

BCMDCMD.exe는 7가지 주요 명령을 지원한다.

| 명령                           | 기능                             | 연결 방식 |
| ------------------------------ | -------------------------------- | --------- |
| `print`                        | 이미지+설정 → ARX4 데이터 생성   | 드라이버  |
| `send`                         | ARX4 데이터 → 프린터 전송 (인쇄) | 드라이버  |
| `extract`                      | ARX4 데이터 → 설정/이미지 추출   | 로컬      |
| `Circulation` / `AutoCleaning` | 화이트 잉크 순환 / 자동 클리닝   | LAN 필수  |
| `PrintDisable` / `PrintEnable` | 프린터 인쇄 버튼 비활성화/활성화 | LAN 필수  |
| `MenuLock` / `MenuUnlock`      | 프린터 설정 메뉴 잠금/해제       | LAN 필수  |
| `status`                       | 프린터 상태 CSV 출력             | LAN 필수  |
| `getlog`                       | 프린터 로그 파일 출력            | LAN 필수  |
| `picklog`                      | 로그 파일에서 이력 CSV 추출      | 로컬      |

## 3. 명령어 상세

### 3-1. `print` - ARX4 데이터 생성

```
BCMDCMD.exe print -X <XML파일> -I <이미지파일> -P <프린터> -A <ARX4파일> -L <위치> -S <크기> -R <배율> -W <화이트>
```

#### 인자 (Arguments)

| 인자          | 필수 | 설명                                                                      |
| ------------- | ---- | ------------------------------------------------------------------------- |
| `-X ####`     | O    | 인쇄 설정 XML 파일 경로                                                   |
| `-I ####`     | O    | 인쇄할 이미지 파일 (PNG만 지원)                                           |
| `-P ####`     | △    | 프린터 지정 (-A와 상호 배타)                                              |
| `-A ####`     | △    | 출력 ARX4 파일명 (-P와 상호 배타)                                         |
| `-L nnnnnnnn` | O    | 인쇄 위치 (8자리, 앞4자리=좌측여백, 뒤4자리=상단여백, 단위 0.1mm)         |
| `-S nnnnnnnn` | △    | 인쇄 크기 (8자리, 앞4자리=너비, 뒤4자리=높이, 단위 0.1mm, -R과 상호 배타) |
| `-R nnnn`     | △    | 배율 (4자리, 단위 0.1%, -S와 상호 배타)                                   |
| `-W n`        | -    | 0: RGB(255,255,255)를 투명으로, 1: 화이트로 해석                          |

#### XML 설정 요소 (`<GTOPTION>` 하위)

| 요소                | 값 범위    | 설명                                                                    |
| ------------------- | ---------- | ----------------------------------------------------------------------- |
| `<szFileName>`      | 문자열     | ARX4 저장 파일명 (보통 -A 인자로 대체)                                  |
| `<uiCopies>`        | 1~999      | 인쇄 매수                                                               |
| `<byMachineMode>`   | 0          | 기기 모드 (0: B-422)                                                  |
| `<byPlatenSize>`    | 0~4        | 플래튼 크기 (0: 16x21, 1: 16x18, 2: 14x16, 3: 10x12, 4: 7x8)            |
| `<byInk>`           | 0~3        | 잉크 조합 (0: Color Only, 1: White Only, 2: Color+White, 3: Black Only) |
| `<bEcoMode>`        | true/false | 에코 모드 (byInk=2일 때만 유효)                                         |
| `<byResolution>`    | 1          | 해상도 (1: 1200x1200 dpi)                                               |
| `<byHighlight>`     | 1~9        | 화이트 두께/밝기 (byInk=1,2일 때)                                       |
| `<byMask>`          | 1~5        | 컬러 잉크 베이스 화이트 양 (byInk=1,2일 때)                             |
| `<byInkVolume>`     | 1~10       | 컬러 잉크 볼륨 (byInk=0일 때)                                           |
| `<byDoublePrint>`   | 0~3        | 더블 프린팅 (byInk=0일 때, 0: 1회, 1~3: 2패스)                          |
| `<bMaterialBlack>`  | true/false | 배경 검정색 사용 (byInk=2일 때)                                         |
| `<bMultiple>`       | true/false | 컬러 멀티패스 인쇄 (byInk=0,2일 때)                                     |
| `<bTransColor>`     | true/false | 투명색 사용 (byInk=1,2일 때)                                            |
| `<colorTrans>`      | 숫자       | 투명색 RGB (BGR 순서 16진수 → 10진수 변환)                              |
| `<byTolerance>`     | 0~50       | 투명색 그라데이션 범위                                                  |
| `<byMinWhite>`      | 1~6        | 최소 화이트 볼륨 (byInk=2일 때)                                         |
| `<byChoke>`         | 0~10       | 초크 폭 - 화이트 잉크 범위 축소 (byInk=2일 때)                          |
| `<bPause>`          | true/false | 화이트/컬러 간 일시정지 (byInk=2일 때)                                  |
| `<bySaturation>`    | 0~40       | 채도                                                                    |
| `<byBrightness>`    | 0~40       | 밝기                                                                    |
| `<byContrast>`      | 0~40       | 대비                                                                    |
| `<iCyanBalance>`    | -5~5       | 컬러 밸런스: 시안                                                       |
| `<iMagentaBalance>` | -5~5       | 컬러 밸런스: 마젠타                                                     |
| `<iYellowBalance>`  | -5~5       | 컬러 밸런스: 옐로                                                       |
| `<iBlackBalance>`   | -5~5       | 컬러 밸런스: 블랙                                                       |
| `<bUniPrint>`       | true/false | 단방향 인쇄 강제                                                        |

### 3-2. `send` - ARX4 데이터 프린터 전송

```
BCMDCMD.exe send -A <ARX4파일> -P <프린터>
```

| 인자      | 필수 | 설명             |
| --------- | ---- | ---------------- |
| `-A ####` | O    | 전송할 ARX4 파일 |
| `-P ####` | O    | 대상 프린터명    |

### 3-3. `extract` - ARX4 데이터에서 설정/이미지 추출

```
BCMDCMD.exe extract -A <ARX4파일> -X <XML파일> -I <이미지파일> -S <이미지크기>
```

| 인자          | 필수 | 설명                                            |
| ------------- | ---- | ----------------------------------------------- |
| `-A ####`     | O    | 소스 ARX4 파일                                  |
| `-X ####`     | △    | 설정 출력 XML (-X 또는 -I 중 하나 이상 필수)    |
| `-I ####`     | △    | 이미지 출력 PNG (-X 또는 -I 중 하나 이상 필수)  |
| `-S nnnnnnnn` | -    | 출력 이미지 최대 크기 (픽셀 단위, -I 지정 시만) |

추출된 XML의 루트 요소는 `<GTDATA>`이며, 추가로 다음 정보가 포함된다:

- `<szJobName>`: 작업명
- `<szDateTime>`: 생성 일시 (MM/DD/YYYY hh:mm)
- `<szPlatenSize>`: 플래튼 크기 문자열
- `<byTime>`: 인쇄 시간 배율 (4의 배수)
- `<iWhiteness>`: 화이트 잉크 소비 추정치 (200~600, 단위 1%)
- `<iInkColor>`: 컬러 잉크 소비량 (단위 0.01cc)
- `<iInkWhite>`: 화이트 잉크 소비량 (단위 0.01cc)
- `<iCopies>`: 인쇄 매수

### 3-4. 프린터 제어 명령 (LAN 전용)

```
BCMDCMD.exe <명령> -P <프린터>
```

| 명령           | 기능                      |
| -------------- | ------------------------- |
| `Circulation`  | 화이트 잉크 순환 실행     |
| `AutoCleaning` | 자동 클리닝 실행          |
| `PrintDisable` | 프린터 인쇄 버튼 비활성화 |
| `PrintEnable`  | 프린터 인쇄 버튼 활성화   |
| `MenuLock`     | 설정 메뉴 잠금            |
| `MenuUnlock`   | 설정 메뉴 해제            |

> LAN 연결 프린터만 지원. USB 연결 프린터는 지원하지 않음.

### 3-5. `status` - 프린터 상태 CSV 출력 (LAN 전용)

```
BCMDCMD.exe status -P <프린터> -S <상태CSV> -O <옵션CSV> -M <유지보수CSV>
```

3종류의 CSV를 출력할 수 있으며, `-S`, `-O`, `-M` 중 하나 이상 필수:

#### Status CSV (`-S`)

- 프린터 상태 (16진수 OR 비트마스크: 0x01=초기화중, 0x02=대기, 0x04=준비, 0x08=인쇄중, 0x10=메뉴활성, 0x20=에러정지)
- 에러 코드: Fatal Error, Fatal Error2, Usual Error, Wait OK, Wait OK2, Warning (각 최대 20개)

#### Option CSV (`-O`)

- Print Direction (0=양방향, 2=단방향, 6=자동)
- Back Light (254/200/150)
- Auto Screen Erase Time (0~4)
- Menu TimeOut (0~2)
- Speaker Volume (0~7)
- Temperature (0=화씨, 1=섭씨)
- Low Temperature Mode (0=활성, 1=비활성)
- Auto Cleaning 설정 (간격 1~24, 타이밍)
- Flushing 설정 (간격, 볼륨)
- 네트워크 설정 (DHCP, Host Name, IP, Subnet, Gateway)
- Print Button Disable, Menu Lock, Auto Job Delete, Rcvng Prnt Setting

#### Maintenance CSV (`-M`)

- 총 인쇄 횟수
- 펌웨어 버전 (Main, Maintenance, LCD, Head, Message, Font, Test Print)
- 잔여 잉크량: K, C, M, Y, W12, W34 (0~6, '-'=미장착)
- Auto Cleaning 이력 (마지막 날짜, 이후 인쇄 수)
- White Ink Circulation / Refilling 마지막 일시
- Cap/Wiper Cleaning 이력
- Nozzle Guard Cleaning 이력
- Wiper Cleaner 교체 이력 (Color/White)
- Flushing Foam 교체 이력 (Color/White)

### 3-6. `getlog` - 프린터 로그 파일 출력 (LAN 전용)

```
BCMDCMD.exe getlog -P <프린터> -L <로그파일>
```

프린터에서 바이너리 로그 파일을 다운로드한다. 이 파일은 `picklog` 명령으로 분석 가능.

### 3-7. `picklog` - 로그 파일에서 이력 CSV 추출

```
BCMDCMD.exe picklog -L <로그파일> -P <인쇄이력CSV> -O <운영이력CSV> -M <유지보수이력CSV> -S <시작일시> -E <종료일시>
```

| 인자      | 필수 | 설명                                         |
| --------- | ---- | -------------------------------------------- |
| `-L ####` | O    | 입력 로그 파일                               |
| `-P ####` | △    | 인쇄 이력 CSV 출력 (-P, -O, -M 중 하나 이상) |
| `-O ####` | △    | 운영 이력 CSV 출력                           |
| `-M ####` | △    | 유지보수 이력 CSV 출력                       |
| `-S ####` | -    | 시작 일시 (dd/mm/yyyy [hh:mm:ss])            |
| `-E ####` | -    | 종료 일시 (dd/mm/yyyy [hh:mm:ss])            |

#### 인쇄 이력 CSV (`-P`)

- clock: 인쇄 완료 시각
- name: 작업명
- time: 소요 시간
- purgeK/purgeMCY/purgeW1/purgeW234: 헤드 클리닝 횟수
- inkCMYK: 컬러 잉크 소비량 (cc)
- inkW: 화이트 잉크 소비량 (cc)
- inkAll: 총 잉크 소비량 (cc)
- machine: 프린터명
- platen: 플래튼 크기
- select ink: 잉크 조합
- resolution/highlight/mask/volume/double 등 인쇄 설정값
- temperature/humidity: 인쇄 시 환경 정보

#### 운영 이력 CSV (`-O`)

- clock / status / parameter 3열 구조
- 주요 이벤트: Power ON/OFF, Print, Test Print, Idle(Init/End/Stop), Error, Warning

#### 유지보수 이력 CSV (`-M`)

- clock / status / parameter1 / parameter2 4열 구조
- 클리닝 종류: Normal, Powerful, Super, Auto, Tube, White Head
- 잉크 설치: Initial, Cleaning 후, Head 교체 후, Pause 복구, Filter 교체 후
- White Ink Refilling (Normal/Powerful)
- Closure Cleaning, Exhaust Cleaning
- Head Replacement (Color/White)
- 설정 변경 이력: Print Direction, Backlight, Flushing, Auto Cleaning 등
- 펌웨어 업데이트, 위치 조정, V-rating, Firing 조정 등

## 5. Ver.2.6.0 변경사항

1. `<byInk>` 요소에 **Black Ink Only** 옵션 추가 (값: 3)
2. `<bEcoMode>` 요소 추가 - 에코 모드 설정 (base layer 화이트 잉크량 자동 조정)
3. 유지보수 CSV의 일부 항목이 "헤드 클리닝 횟수"에서 **"인쇄 횟수"**로 변경
   - Replace Wiper Cleaner for Color/White (Start Timing)
   - Replace Flushing Foam for Color/White
4. 잉크 소비량 단위 보정: 0.001cc → **0.01cc**
5. 인쇄 이력 CSV 잉크 소비량 단위: ml → **cc**

## 6. 연동 모듈 구현 시 고려사항

### 6-1. 플랫폼 제약

- BCMDCMD.exe는 **Windows 전용** 실행 파일
- BCMD 프린터 드라이버가 설치된 환경에서만 `print`/`send` 명령 사용 가능
- `Circulation`, `AutoCleaning`, `status`, `getlog` 등은 **LAN 연결 필수**

### 6-2. 핵심 워크플로우

```
[이미지(PNG)] + [설정(XML)]
        ↓ print
    [ARX4 데이터]
        ↓ send
    [프린터 출력]
```

### 6-3. 데이터 형식

- **입력 이미지**: PNG만 지원
- **인쇄 데이터**: ARX4 포맷 (B-Corp 독자 형식)
- **설정 파일**: XML
- **상태/이력**: CSV

### 6-4. 연동 모듈 주요 기능 후보

1. **인쇄 작업 관리**: XML 설정 생성 → ARX4 변환 → 프린터 전송
2. **프린터 상태 모니터링**: status 명령으로 상태/에러/잉크잔량 주기적 조회
3. **유지보수 관리**: 클리닝/잉크순환 자동 스케줄링, 유지보수 이력 추적
4. **인쇄 이력 분석**: 로그 다운로드 → 이력 추출 → 잉크 소비/환경 데이터 분석
5. **프린터 제어**: 인쇄 버튼 활성화/비활성화, 메뉴 잠금 등 원격 제어

### 6-5. 에러 핸들링

- 모든 명령은 리턴 코드로 결과를 반환하므로, `%ERRORLEVEL%` 기반 에러 처리 필요
- 에러 코드 범위별로 분류하여 사용자 친화적 메시지 변환 권장
- 프린터 연결 에러(-2401, -2701 등)에 대한 재시도 로직 필요
